EduXAI Studio - Teacher Manual (Step by Step)

Use this workflow to generate one conceptual exercise core per request, review XAI quality, and export a student-ready package with controlled UDL variants.

Step 1. Configure generation

  1. Pick Provider mode: Direct API or Manual AI prompt.
  2. Choose Exercise type (required).
  3. Set teacher-locked invariants: Learning objective, Bloom, Difficulty.
  4. Select Exercise output language.
  5. Optional: enable UDL variation (max 3 variants).

Gemini API key (free tier)

If you use Direct API, create a Gemini API key in Google AI Studio. Google offers a free tier with usage limits that can work for classroom pilots.

  • Create and manage keys: Gemini API key guide
  • Check current limits/pricing: Gemini API pricing and limits

Recommendation: if your quota is exhausted, switch to Manual AI prompt mode temporarily.

Step 2. Generate and review

  1. Paste or upload source content and click Generate with AI.
  2. Review in Exercise Explorer: design, preview, pedagogical summary, technical XAI detail, validation.
  3. Mark exercises as reviewed once pedagogically approved.

Tip: if the model output is incomplete, Studio applies local minimum XAI auto-repair before final validation.

Step 3. Validate before export

  • Check Validation Results and resolve critical errors first.
  • Inspect warnings for semantic drift across UDL variants.
  • Use XAI Insights for a quick transparency summary.

Step 4. Export

  • Export teacher project: complete JSON with XAI and editing metadata.
  • Export for students: package optimized for EduXAI-Visor.
  • In manual UDL export mode, choose one variant strategy before exporting.
