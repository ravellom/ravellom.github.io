EduXAI Studio - Manual docente paso a paso

Usa este flujo para generar un nucleo de ejercicio por solicitud, revisar la calidad XAI y exportar un paquete final para estudiantes con variantes UDL controladas.

Paso 1. Configura la solicitud

  1. Selecciona modo de proveedor: API directa o prompt manual.
  2. Elige tipo de ejercicio (obligatorio).
  3. Define invariantes docentes: objetivo, Bloom y dificultad.
  4. Define idioma de salida del ejercicio.
  5. Opcional: activa variacion UDL (maximo 3 variantes).

API key de Gemini (free tier)

Si usas API directa, crea una API key en Google AI Studio. Google ofrece un free tier con limites de uso que puede servir para pilotos docentes.

  • Crear y gestionar claves: Guia de API key de Gemini
  • Consultar limites/precios vigentes: Precios y limites de Gemini API

Recomendacion: si se agota la cuota, usa temporalmente el modo prompt manual.

Paso 2. Genera y revisa

  1. Pega o sube contenido fuente y pulsa Generar con IA.
  2. Revisa en Explorador de ejercicios: diseno, vista previa, resumen pedagogico, detalle XAI tecnico y validacion.
  3. Marca como revisados los ejercicios aprobados pedagogicamente.

Nota: si la salida llega incompleta, Studio aplica auto-reparacion local minima de campos XAI antes de la validacion final.

Paso 3. Valida antes de exportar

  • Consulta Resultados de validacion y corrige primero los errores criticos.
  • Revisa advertencias de deriva semantica entre variantes UDL.
  • Usa XAI Insights para el resumen de transparencia.

Paso 4. Exporta

  • Exportar proyecto docente: JSON completo con XAI y metadatos de edicion.
  • Exportar para estudiantes: paquete optimizado para EduXAI-Visor.
  • En modo manual UDL, selecciona estrategia de variantes antes de exportar.

Que campos XAI debe incluir cada ejercicio?

Cada ejercicio debe incluir un bloque xai completo:

why_this_exercisepedagogical_alignmentcontent_selectiondesign_rationalefairness_and_riskhuman_oversightquality_of_explanationuncertaintycounterfactualtrace
